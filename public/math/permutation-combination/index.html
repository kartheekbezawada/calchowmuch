Formula Guide

Use nPr = n! / (n - r)! when order matters and nCr = n! / (r!(n - r)!) when order does not. Factorial mode uses n! = n × (n - 1) × ... × 1 and acts as the base building block for both counting formulas.

FAQ

What is the simplest way to tell nPr from nCr?

If swapping positions changes the outcome, use a permutation. If swapping does nothing, use a combination.

Why does factorial mode hide r?

Factorial only needs one input because it counts all arrangements of the full set from n down to 1.

Why is n limited to 170?

Values above 170 become too large for the current exact-number handling and can overflow practical display limits.

Why is nPr larger than nCr?

Permutations count every order separately, while combinations collapse those repeated orderings into one selection.

What does 0! equal?

Zero factorial equals 1 by definition, which keeps many counting formulas consistent.

Does the route round the answer?

No. Results stay as exact integers so you can use them directly in later probability or counting work.
