## 3. Link Objective

### Core Goal
Keep users circulating between complementary debt-planning jobs:
- one-card payoff
- minimum-payment reality
- consolidation comparison
- balance-transfer comparison
- multi-debt payoff planning

### Page Role
`Debt Payoff Calculator` is the flagship strategy page in this loop.

It should function as:
- the broad action-planning route
- the comparison hub
- the page that links users outward to adjacent specialized tools

## 4. Linking Aggression Level

This page uses a very aggressive internal-link model.

That means links are required in:
- above-the-fold related links
- contextual body copy
- strategy-comparison sections
- example-scenario sections
- FAQ answers
- next-step blocks after results

Aggressive does not mean sloppy. Each link must still feel context-appropriate and useful.

## 6. Contextual Link Placement

### In the Main Guide
Use contextual links where the user naturally needs a different calculation job.

#### To Credit Card Payment Calculator
Use when discussing:
- a single card
- one-balance payoff cases
- how one-card repayment differs from a full debt plan

#### To Credit Card Minimum Payment Calculator
Use when discussing:
- minimum-only payments
- the cost of sticking to issuer minimums
- why debt payoff strategy matters

#### To Credit Card Consolidation Calculator
Use when discussing:
- combining debts
- changing the structure of repayment
- comparing payoff strategy against a consolidation path

#### To Balance Transfer Credit Card Calculator
Use when discussing:
- promotional-rate alternatives
- lower-APR restructuring
- whether transfer economics beat a pure payoff strategy

## 7. Strategy-Comparison Section Linking

The snowball vs avalanche section must include contextual links where helpful.

### Required Jobs

- compare payoff strategy against balance transfer

## 8. Worked-Scenario Link Placement

Each named worked scenario may include targeted next-step links.

### Scenario Linking Rules
- Use scenario-specific anchors tied to the debt profile
- Do not use identical generic anchors in all scenarios
- Favor the most relevant specialized calculator for each scenario

### Example Mappings
- Single-card-heavy scenario -> Credit Card Payment Calculator
- Minimum-payment-trap scenario -> Credit Card Minimum Payment Calculator
- Restructure-or-transfer scenario -> Consolidation or Balance Transfer calculator

## 9. FAQ Link Placement

FAQ answers should include links when the answer naturally points to a different tool.

Appropriate FAQ expansions:
- “What if I only make minimum payments?”
- “Should I consolidate instead?”
- “Would a balance transfer help?”
- “What if I only have one card?”

FAQ links should:
- answer the question first
- link second
- avoid sounding promotional

## 11. Anchor Strategy By User Job

### Minimum-Only Cost
Preferred anchor themes:
- see what minimum-only payments cost
- compare against minimum-payment payoff
- test the minimum-payment trap

### One-Card vs Multi-Debt Plan

- run the one-balance version

### Consolidation vs Payoff
Preferred anchor themes:
- compare payoff vs consolidation
- test whether debt consolidation helps
- check a consolidation scenario

### Balance Transfer vs Payoff Strategy
Preferred anchor themes:
- compare payoff vs balance transfer
- test whether a transfer saves more
- check a lower-APR transfer option

## 12. Anchor Quality Rules

- Anchors must be intent-led and task-led.
- Anchors must not be generic filler.
- Avoid repeated anchor + target pairs within the same section.
- Vary phrasing across the page while keeping the job clear.
- Do not use “click here”, “learn more”, or “related calculator” anchors.

## 13. Circular Back-Link Expectations

The following adjacent routes should eventually link back into `Debt Payoff Calculator`:

### From Credit Card Payment Calculator
- when the user appears to need a multi-debt strategy instead of a single-card estimate

### From Credit Card Minimum Payment Calculator
- when the content explains moving beyond minimum-only repayment

### From Credit Card Consolidation Calculator
- when the user should compare consolidation against direct payoff strategies

### From Balance Transfer Credit Card Calculator
- when the user should compare promotional transfer economics against structured debt payoff

## 14. Link Quality Constraints

- Aggressive linking is required, but the page must still feel deliberate and trustworthy.
- Do not dump all target links into every section.
- Do not let internal links overtake the debt-free-date-first hierarchy.
- Keep links tied to genuine user decisions and comparisons.
