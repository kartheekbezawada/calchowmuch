# REQ-20260208-022 — Energy-Based Nap Selector (Time & Date)

## Status: COMPLETE

- Type: Calculator
- Change Type: New Calculator + SEO
- Priority: HIGH
- SEO Impact: YES
- New route (must be added): `/time-and-date/energy-based-nap-selector/`

## 1) Objective

Publish an “energy-based” nap selector calculator that recommends a nap duration (15/25/90 minutes) and returns wake-up time + alternatives, with strict UI interaction rules.

## 2) Scope

In-scope:
- New calculator page (Time & Date section).
- Deterministic mapping logic and UI.
- Explanation Pane per UI-EXP-PANE-STD.
- SEO package per SEO_RULES.
- Navigation + sitemap coverage (DOC-SITEMAP-1..5).

## 3) Calculation Rules

Inputs:
- Start time (time of day)
- Goal (segmented buttons): Quick / Strong / Full

Primary mapping (deterministic):
- Quick → 15 minutes
- Strong → 25 minutes
- Full → 90 minutes

Outputs:
- Primary recommendation card: duration, wake-up time, short rationale.
- Two alternative options with wake-up times.

Time-of-day warning rules:
- Categorize start time into Day / Afternoon / Night buckets.
- If the selection implies a long nap late at night, show a warning.

Override rule:
- If goal is inferred as “Full” and the start time is Night, downgrade recommendation to 25 minutes.
- If the user explicitly selects Full, do not override; show the warning instead.

## 4) Calculation Pane Interaction Contract

- Goal must be a segmented button group (no dropdown) (UI-2.5, ISS-UI-FDP-009/010).
- Default goal at page load must be defined and implemented (UITGL-5).
- After initial baseline, input edits and goal toggles must not recompute until Calculate is clicked (UI-2.6, ISS-UI-FDP-008).
