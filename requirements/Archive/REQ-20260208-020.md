# REQ-20260208-020 — Work Hours Calculator (Time & Date)

## Status: NEW

- Type: Calculator
- Change Type: Explanation + SEO
- Priority: HIGH
- SEO Impact: YES
- Route (must remain live): `/time-and-date/work-hours-calculator/`

## 1) Objective

Standardize the Work Hours Calculator explanation pane (universal structure + dynamic tables) and upgrade SEO/schema, without changing the existing calculation pane layout.

## 2) Scope

In-scope:
- Explanation Pane rewritten to UI-EXP-PANE-STD.
- Metadata + JSON-LD schema per SEO_RULES.

Out of scope:
- Changing calculation pane layout/labels.
- Introducing dropdowns (UI-2.5).
- Changing compute logic unless required for correctness.

## 3) Calculation Expectations (Behavioral)

The calculator must continue to support:
- Single shift and split shift workflows (if currently present).
- An unpaid break input that reduces paid time.
- Overnight shifts via an explicit “ends next day” control.

Definitions:
- Gross duration = end − start (+24 hours if ends-next-day)
- Paid duration = gross − unpaid break

## 4) Calculation Pane Interaction Contract

- Results and explanation dynamic values must not change on input edits until Calculate click after initial baseline (UI-2.6, ISS-UI-FDP-008).

## 5) Explanation Pane Requirements

Must follow UI-EXP-PANE-STD with:
- Dynamic Summary
- Scenario Summary table
- Results table
- Explanation text
- Exactly 10 FAQs (EXP-FAQ-1) with FAQPage JSON-LD mirroring verbatim

Tables must comply with UTBL-STRUCT-* and UTBL-BORDER-*.

## 6) SEO Requirements

- Canonical must be `https://calchowmuch.com/time-and-date/work-hours-calculator/` (SEO_RULES P1.3).
- JSON-LD: WebPage, SoftwareApplication, BreadcrumbList, FAQPage (SEO_RULES P2.*).

## 7) Acceptance Criteria

- No calculation pane layout drift (UI-3.1, UI-3.2).
- Explanation pane meets UI-EXP-PANE-STD with 10 FAQs.
- SEO P1/P2/P5 passes for this route.

## 8) Required Tests

- SEO: P1/P2/P5.
- E2E: Calculate-only recompute check if wiring changes.
