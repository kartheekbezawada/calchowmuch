# REQ-20260207-009 — ISS-001 Global Shell Layout Stability Fix (All Calculator Pages)

## Status: NEW

- Type: Calculator Platform
- Change Type: Layout/CSS + Test Stabilization
- Priority: HIGH
- SEO Impact: YES
- Related tests: `ISS-001` (mandatory), scoped E2E, SEO validation per governance

## 1) Objective

Fix shared shell layout drift that causes ISS-001 failures across calculators.

Constraints:
- Keep MPA architecture unchanged.
- No calculator formula/business-logic changes.

## 2) Scope

In-scope:
- Shared shell CSS only (left-nav, center-column, ads-column, page container).
- ISS-001 test stabilization for deterministic measurement/snapshot behavior.

Out of scope:
- Any calculator-specific UI changes.
- Any calculator math/logic changes.
- Any new pages, navigation model changes, or SPA routing.

## 3) Required Implementation Details

### 3.1 Shared shell layout stability

- Enforce fixed shell height behavior across navigation.
- Prevent pane height/position shifts during calculator/category navigation.
- Keep scrollbars stable: use `overflow-y: scroll` (not `auto`) and `scrollbar-gutter: stable` where applicable.
- Remove/avoid transitions/animations that cause layout movement in shell containers.

### 3.2 ISS-001 test stabilization

- In the ISS test flow, wait for font readiness before any dimension/screenshot assertions.
- Measurements/snapshots must be taken only after the shell is in its stable layout state.

## 4) Acceptance Criteria

- `npm run test:iss001` passes fully.
- No new failures in affected calculator E2E routes.
- If intentional visual changes occur, snapshot updates are explicitly recorded.
