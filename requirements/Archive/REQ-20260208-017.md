## 1) Objective

Harden the Wake-Up Time Calculator’s calculation logic and standardize the explanation pane + SEO package while preserving the calculator shell layout.

## 2) Scope

In-scope:
- Compute logic for recommended wake-up times.
- Explanation Pane rewritten to the universal standard (UI-EXP-PANE-STD).
- Metadata + JSON-LD schema per SEO_RULES.

Out of scope:
- Any shell layout changes (UI-3.1, UI-3.2).
- Dropdown controls (UI-2.5).

## 3) Calculation Rules

Constants:
- Sleep cycle length = 90 minutes
- Fall-asleep buffer = 15 minutes

Inputs:
- Bedtime (date + time), user-provided.

Computation:
- Sleep start time $S = B + 15\text{ minutes}$
- For each cycle count $c$ (integer), compute wake time:
  - $W_c = S + 90c\text{ minutes}$

Outputs:
- Display a list of recommended wake-up times for cycle counts $c \in \{4, 5, 6\}$.
- Optionally also show $c=3$ and $c=7$ as “earlier/later” options if the existing UI supports it.

## 4) Calculation Pane Interaction Contract

- The page must have an explicit Calculate CTA.
- After the initial page-load baseline, editing inputs must not change results or explanation dynamic values until Calculate is clicked (UI-2.6, ISS-UI-FDP-008).
